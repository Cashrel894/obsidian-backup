为了创建一个**理性计划代理**，我们需要数学化地描述代理所处的环境。

## Search Problem
**搜索问题**(Search Problem)：由代理的当前状态出发，如何以最佳的方式到达满足目标的新状态？

一个搜索问题由以下要素组成：
- **状态空间**(State Space)：已知世界中所有可能状态的集合。
- 每个状态可能行动的集合。
- **转移模型**(Transistion Model)：当前状态 ⨯ 行动 -> 下一状态的映射，表示行动对状态的影响。
- **行动代价**(Action Cost)：从一个状态转移到另一状态的代价。
- **初始状态**(Start State)：代理最初的状态。
- **目标测试**(Goal Test)：状态 -> Boolean 的映射，表示判断状态是否为目标状态。

基本上，要解决搜索问题，我们会先从初始状态出发，再使用行动集合、转移模型和行动代价探索状态空间，直到到达目标状态，从而确定从初始状态到目标状态的**路径**（或**计划**，plan）。而我们考虑可能状态的**顺序**，则称为**策略**(Strategy)。

## Search State
**搜索状态** (Search State) 与**世界状态**(World State) 不同：
- 世界状态包含已知状态的所有信息。
- 搜索状态则只包含对于**计划**必要的信息。

例如在吃豆人游戏中，如果要搜索从一个位置到另一位置的最短路径，那么搜索状态就只包含当前位置的坐标；而要搜索从一个位置出发，吃完地图上所有豆子的最短方案，那么搜索状态不仅包括当前位置坐标，还包括剩余豆子的坐标集合。
