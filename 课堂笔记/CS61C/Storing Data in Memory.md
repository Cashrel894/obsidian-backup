#CS61C 
一个 8-bit 的内存块被称为**字节**。而在 32 位 RISC-V 中，1 个字长=4 个字节（1 word = 4 bytes）

因此，32 位 RISC-V 中一个寄存器恰好存储一个字，32 个寄存器可以存储 32 个字（即 32 * 4 = 128 bytes）

在小端架构中，字地址指的是字中最低位字节的地址。

而在 RISC-V 中，将内存信息加载到 [[Registers]] 有如下方式：
```
lw x10,12(x15) #Reg x10 gets A[3]
```
其中：
- lw = load word (to x10 from x15 with offset of 12)
- x15 存放了指向数组 `A` 的指针。
- 12 指的是相对于 `A[0]` 的**字节**偏移量。偏移量必须为常数。

```
sw x10,40(x15) #A[10] gets Reg x10
```
其中：
- sw = save word (from x10 to x15 with offset of 40)
- 其他逻辑和 lw 差不多

除此之外，还有操作字节的 lb 和 sb，语法类似，操作的对象是寄存器的**最低位字节**。其中，当 lb 时，会自动用**符号位**填充寄存器的高位。

lbu 则是 lb 的无符号变种，用于读取无符号的数据，则会直接使用 0 填充。

lh sh lhu shu 逻辑相似，只不过是操作**半个字**（**h**alf word）。
